Sample: SqlBrowse

Summary:

The purpose of this sample is to illustrate the usage of mixed ISAM and SQL code when using the PostgreSQL server. In fact, the sample uses an XbpBrowse() to view an SQL result set of an ISAM emulating table on the Postgres server. This table is created via an upsize of the customers.dbf ISAM table prior to running the example. Also shown in this example is how a record in the SQL result can be updated in the ISAM emulating table using ISAM code.

Files:

  • main.prg: Entry point of the application. Implements form creation and application startup.
  • upsize.prg: Verifies if a database and table is available at a remote server. Implements the upsize process.
  • form.prg: XbpDialog-based browse form and the edit form.
  • smartform.prg: XbpHTMLWindow-based generic form class
  • northwind-customers.upsize: The upsize definition used to upsize the sample data to the SQL server.
  • resource.arc: Resource file defining the application manifest.

Description:

All navigational features required by the XbpBrowse() class, such as skip blocks and the usage of the percentage-based functions DbPosition() and DbGoPosition(), are supported for all SQL result sets. Therefore, the XbpBrowse() class can be used consistently for DBF tables as well as ISAM tables on the PostgreSQL server. In addition, the XbpBrowse() class can be used for result sets retrieved using the SQL SELECT command. In all these cases, full navigational support is provided.

In addition, this sample also shows the following concepts:

  • Testing for database and table existence using DacInfo()
  • Programmatic creation of a database on the PostgreSQL server
  • Programmatic usage of the DbfUpsize mechanism
  • Execution of SQL statements using the SQLStatement class
  • Usage of the XbpHTMLWindow() to create HTML based forms

Requirements:

  • Xbase++ 2.00.850 or higher.
  • Installed PostgreSQL server on localhost. Alternatively, the connection parameters in the .upsize file and the connection string in main.prg must be adapted to your infrastructure setup.

Keywords:

SQL, XbpBrowse, Postgres, Upsize, DbPosition(), DbGoPosition()
